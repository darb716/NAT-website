Port Address Translation

If you are here:

I am going to assume that you already understand NAT and how NAT works. As well as understanding what Ports are, and what common ports are typically used. If you need a refresher on what ports are I suggest This Website

DNS Request Created
Notice to go to a website we do not use IP's we use Hostnames. This name request is sent to the DNS server.
IP Requested
The DNS server recieves the DNS request from the client
DNS response
The DNS server responds with the IP address that matches the domain name.
Packet Generation
Now that we have the IP address the packet can be generated.
Packet movement 1
The packet begines moving to the router due to 8.8.8.8 not being on the same subnet.
Packet port translated
When the packet is recieved by the router the source port is changed to a different port that is not currently being used. The router then adds the translation that has occurred to an internal database so it can retranslate the response.
Packet Delivered
When the packet is recieved by google the same thing is done on their end. We will skip this for simplicity.
Packet Redirection
When the packet leaves google, it has a new source and destionation port and IP address.
Packet Retranslation
When the router recieves the packet it Re-translates the destination port to be the same as the original source in it's database.
Packet Final Movement
The re-translated packet moves exactly as expected from here on out.
Packet Arrival
The packet is recieved on the port that was originally opened with the source that was expected.

After action report:

How was this right?

This is basically how networks work in the real world when moving things from your house to the internet.

How was this wrong?

Things get more complicated when you start dealing with inbound traffic and Bi-directional traffic. However that is extreemly complicated when it comes to explaining with simply text and a slideshow.

Where to next?

Now that you understand how network work on a baisc level I would suggest some practical practice. Wireshark is very good for this, and if you are feeling really brave you can even install PFsense in your house and use it as your router to get more interesting information. This will help you better understand what is going in and out of your home network as well as allow you to play with packet filtering and more complicated concepts.
