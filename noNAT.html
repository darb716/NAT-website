After action report:

How was this right?

This is how IPV4 networks were designed. This is known as end-to-end addressing. This is how networking was originally created. However when they created IPV4 they did not expect us to run out of IP addresses like we have. So to act as a stop gap NAT was created.

How was this wrong?

In the real world, there are different address spaces for public and Private IP's this means that there would never be a device out on the open internet with a 192.168.x.x address. If you would like to learn about the different IP address ranges I would like to refer you to this page for more help on the basic concepts of networking.
